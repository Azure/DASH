# SAI-Thrift Server

Builds a SAI-thrift server for DASH bmv2 switch.

See:
*  [Compile P4 Code](#compile-p4-code)
*  [Build libsai.so adaptor library](#build-libsaiso-adaptor-library)

## Summary of problems:
* *Minor* - SAI/meta/Makefile needs to pass `-e` flag to generate extensions; this can be manually hacked.
* **URGENT** DASH SAI code generator has type conversion errors and is missing baseline SAI library functions
# Quick instructions

* It uses a container `chrissommers/dash-saithrift-bldr:220624` which is built according to 
[Dockerfile.saithrift](dockerfiles/Dockerfile.saithrift) which contains [prerequisites](https://github.com/opencomputeproject/SAI/blob/088627dd90c3420daf96d294c661b4a152afb01e/ptf/SAI_PTF_user-guide.md#install-prerequisites)
* Top-level [Makefile](Makefile) spawns a container and calls the next Makefile below
* The detailed commands based on [PTF User Guide](https://github.com/opencomputeproject/SAI/blob/088627dd90c3420daf96d294c661b4a152afb01e/ptf/SAI_PTF_user-guide.md) are in
[DASH/dash-pipeline/SAI/saithrift/Makefile](SAI/saithrift/Makefile)

One time preparation:
```
mkdir workdir && cd workdir
git clone https://github.com/chrispsommers/DASH.git
cd DASH
git checkout chris-saithrift3
git submodule update --init  # one time, after cloning
```
To build:
```
make clean                   # prepare environment
make p4                      # compile P4, generate P4Info
make sai                     # autogen sai headers and libsai.so from P4Info
make sai-thrift-server       # generate sai rpc server and libraries
```
OR, just:
```
make clean all sai-thrift-server
```

It suceeds up to compiling the final Thrift RPC server, where it fails linking to `libsai`. I suspect there are some missing APIs needed in libsai.so. This library is generated from P4 code as described in the main [README](README.md), see the links at the top of this page.
